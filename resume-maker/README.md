# Resume Maker — AI-Powered Application Suite

Give Claude Code a job description. Get back a tailored resume (PDF), cover letter, and interview prep guide — all generated by Claude, no manual steps.

## The Idea

Three documents. One coherent story. Each one building toward the next:

| Document | Purpose |
|---|---|
| **Cover Letter** | Soft opener — creates intrigue, makes them *want* to open the resume |
| **Resume (PDF)** | Adds proof — makes them *want* to schedule an interview |
| **Interview Prep** | Seals the deal — STAR answers, questions to ask, salary negotiation |

## Setup

### 1. Fill in your ABOUTME.md

Open `resume-maker/ABOUTME.md` and replace the example with your real background. **More detail = better output.** Claude selects the most relevant parts for each role.

Include: work history with specific achievements and metrics, education, skills, projects, certifications, talks, publications.

**Keep expanding it** every time you ship something new — it's your living career knowledge base.

### 2. Install rendercv (for PDF generation)

```bash
pip install "rendercv[full]"
```

That's it. No API key setup needed — Claude Code is already authenticated.

---

## Usage — The `/apply` Command

Open Claude Code in this project directory and run:

```
/apply <paste the job description here>
```

Or for long job descriptions, just paste the text directly after `/apply`:

```
/apply
Senior Software Engineer — Platform Team
Stripe

About Stripe
Stripe is a financial infrastructure platform...
[paste the full job posting]
```

Claude will:
1. Read your `ABOUTME.md`
2. Analyze the job description
3. Select and curate your most relevant experience
4. Define a positioning angle (the core narrative for this application)
5. Write the RenderCV YAML resume
6. Render it to PDF with `rendercv`
7. Write a tailored cover letter
8. Write an extensive interview prep guide

Everything lands in `resume-maker/outputs/<company>_<role>_<date>/`.

## Output

```
resume-maker/outputs/
└── stripe_senior-software-engineer_2025-01-15/
    ├── Jane_Doe_CV.yaml       ← Edit and re-render anytime
    ├── Jane_Doe_CV.pdf        ← Ready to send
    ├── cover_letter.md        ← Tailored cover letter
    └── interview_prep.md      ← Your interview playbook
```

### Re-rendering after edits

After tweaking the YAML:

```bash
rendercv render resume-maker/outputs/stripe_.../Jane_Doe_CV.yaml
```

### Available themes

| Theme | Style |
|---|---|
| `classic` | Clean, traditional (default) |
| `sb2nov` | Modern, minimal |
| `engineeringresumes` | Tech-focused |
| `moderncv` | Contemporary |
| `toberecruited` | Bold, modern |

Change the theme in the YAML under `design.theme:`, then re-render.

---

## Tips

- **One run per application.** Even similar roles at different companies need different positioning.
- **Review the YAML before sending.** Claude is good but not infallible — check dates and phrasing.
- **Add your voice to the cover letter.** Claude gives you the structure; personalize the closing in your own words.
- **Say the STAR answers out loud.** The interview prep is tailored to your actual experience — practice it.
- **Update ABOUTME.md constantly.** Add new achievements as they happen, not when you're job hunting.

## Alternative: Run the Python Script Directly

If you want to use this outside of Claude Code (e.g. in a CI pipeline or from another IDE), the `generate_resume.py` script calls the Claude API directly:

```bash
export ANTHROPIC_API_KEY="sk-ant-..."
pip install -r requirements.txt
python generate_resume.py --job example_job.txt
```

But the `/apply` slash command is the intended way to use this — no API key management, no terminal, just paste and go.
